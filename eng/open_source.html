Open Source

Open / Free source softwareSo first, what is free and open-source software the term is a bit important because open-source software and free software are not necessarily the same things, free software doesn't mean open source. Many of us remember freeware on the internet from when we were younger this is software where we have no access to the source code and you can't work with it you can't modify it due to the license. This is not great for science.

Open / Free source software

on the other hand, open source doesn't mean free so for those of you who are producing research software just because you're making your software open source it doesn't mean it has to be free software. And the intersection of these two is often known as free and open-source software or FOSS free open-source software and this just refers to off software there's both open and free.

Open source is simply licensing your work so it can be used how you want. Making your software open source is just a matter of setting out the terms of how you want your software to be used it doesn't mean that you have to give up control of your software, it doesn't mean that you're putting it out to the community forever. It just means you're choosing and saying for yourself setting boundaries on how you want your software to be used. For those who want, you open source also makes it easy for others to remix your software to reuse it to build upon your software add new features if they want and if you're building a project where community is important where you want your software to be used by a lot of people making it open source in such a way that people can reuse and work with can be a really great boost for your software just in terms of visibility and people who want to use it things like that. So why use and promote open source?

Open source is part of the ethics of um scientists and hackers a little bit when you're writing this software when you're working on projects together. A lot of the libraries you use, for Python things like this if you're using different programming languages a lot of py libraries you use will be open source. You're benefiting from a lot of work that other people have done so far, and it's a nice feeling to collaborate and give back now, you've built your software or your project on. Years and years of free work that was produced before, and now you're giving back to the community. It also means distributed innovation, so, when, you're making software open source and free so that other people can work on it, this means that other people will give back contributions they'll report bugs when they have issues running your software, they will tell you new features that they want. You don't have to implement them, but you're able to take all of these good ideas from the community and things like this. Importantly for science open source is easier to review reuse and integrate and.

When we talk about open source and science, if you want your software to be distributed widely to be used in a lot of systems. One of the best things you can do for it is licensing it in such a way that everyone can use it and set it up on their own system things like galaxy

Risks of closed code

there are a lot of risks associated with closed code to when you have closed source software used in science , you can’t review it very easily , the community cant review it easily , only the reviewer of the paper can sometimes and if there are errors/bugs in the software then can be missed for and years and years , And the this can affect the loss of science results downstream .so having your open source reviewable .it’s very important for reproducible ,good quality science, a lot of journals are adopting this and starting to require disclosure or publishing of your source code for all of your software ,this is from scientific perspective,its really a good thing for the community.

But it’s not just for software but also for data for thing like databases that want to make accessible for photo of training materials. all these are options. and if you want people to be able to use them you need to license it.

Fears

One of the common fears people have is that is they publish their software online someone will steal it, but isn’t necessarily true. if someone tries steal your software there is at least a traceable log in GitHub. there is a copy of your software already online and if you already have a community around that, it will be very obvious that someone took it and if you’re are still worried, they publish preprints as way to document that “hey, they were first to write the software” and to really make sure they stake your claim on the software, so don’t worry about just work in the open, it’s better for the community, it’s better for the world, and it’s good for science, publishing and sharing.

Version control

Sharing and publishing, one of the easiest and most effective ways to do, this is through version control. Version control is a fantastic way of share and publish code with others. it gives you whole timeline of the of your software and it make it easy to reuse, contribute and make modifications to your software

Why use version control

• It makes collaboration easy. • Reverts accidents • Merge changes from multiple developer.

Examples of version control system The common ones are Git & Github.com, others are Bitbucket.org, Gitlab.com, Gitea.io. Where to learn? And if you want to learn about Git you can check out this website swcarpentry.Github.io/git-novice/ , • Git is complex. • Start with the parts you need (commit, push, pull). • The rest comes later.
